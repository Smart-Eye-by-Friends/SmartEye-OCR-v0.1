### 📋 슬라이드 3: 기술 스택 선택 근거

#### **Frontend: React 18**

- ✅ **컴포넌트 기반**: 재사용성과 유지보수성
- ✅ **Hooks**: 상태 관리 최적화
- ✅ **Axios**: REST API 통신
- ✅ **실시간 UI**: 분석 진행률 표시

#### **Backend: Spring Boot 3.5**

- ✅ **엔터프라이즈급**: 안정성과 확장성
- ✅ **비동기 처리**: CompletableFuture로 성능 최적화
- ✅ **마이크로서비스**: WebClient로 서비스 간 통신
- ✅ **보안**: CORS, 검증, 예외 처리

#### **AI Service: Python FastAPI**

- ✅ **고성능**: 비동기 처리로 빠른 응답
- ✅ **AI 생태계**: PyTorch, HuggingFace 통합
- ✅ **자동 문서화**: OpenAPI 스펙 자동 생성
- ✅ **독립 배포**: 마이크로서비스로 분리

#### **Infrastructure: Docker + PostgreSQL**

- ✅ **컨테이너화**: 일관된 배포 환경
- ✅ **오케스트레이션**: Docker Compose
- ✅ **관계형 DB**: 복잡한 데이터 관계 지원
- ✅ **확장성**: 수평/수직 스케일링 지원

### 📋 슬라이드 4: 마이크로서비스 아키텍처

#### **🔧 서비스 분리 전략**

**1. Frontend Service (React)**

```
📱 사용자 인터페이스
├── 이미지 업로드 컴포넌트
├── 분석 결과 표시 탭
├── 텍스트 편집기
└── 실시간 진행률 표시
```

**2. Backend Service (Spring Boot)**

```
☕ 비즈니스 로직 처리
├── REST API 엔드포인트
├── 파일 업로드/다운로드
├── 사용자 인증/권한
├── 작업 큐 관리
└── 데이터베이스 연동
```

**3. LAM Service (Python FastAPI)**

```
🧠 AI 분석 전담
├── YOLO 객체 감지
├── 레이아웃 분석
├── 구조화된 결과 생성
└── 모델 버전 관리
```

**4. Database Service (PostgreSQL)**

```
🗄️ 데이터 영속성
├── 사용자 정보
├── 분석 작업 이력
├── 결과 데이터
└── 파일 메타데이터
```

#### **🔄 서비스 간 통신**

- **Frontend ↔ Backend**: HTTP REST API
- **Backend ↔ LAM Service**: WebClient (비동기)
- **Backend ↔ Database**: JPA/Hibernate
- **Backend ↔ External API**: OpenAI API 연동

### 📋 슬라이드 5: AI 파이프라인

#### **🧠 3단계 AI 분석 파이프라인**

**1단계: 레이아웃 분석 (YOLO)**

```
📸 입력 이미지
    ↓
🔍 객체 감지 (YOLO 모델)
    ↓
📊 레이아웃 요소 추출
   ├── 텍스트 영역
   ├── 이미지 영역
   ├── 표 영역
   └── 문제 영역
```

**2단계: 텍스트 추출 (OCR)**

```
📝 감지된 텍스트 영역
    ↓
🔤 Tesseract OCR 처리
    ↓
📄 추출된 텍스트
   ├── 한국어 지원
   ├── 영어 지원
   └── 신뢰도 점수
```

**3단계: AI 해석 (GPT)**

```
📋 구조화된 데이터
    ↓
🤖 OpenAI GPT-4 분석
    ↓
🎯 최종 결과
   ├── 문제별 분리
   ├── 내용 해석
   ├── 설명 생성
   └── JSON 구조화
```

#### **⚡ 성능 최적화**

- **비동기 처리**: 3단계 파이프라인 병렬 실행
- **캐싱**: 모델 로딩 시간 단축
- **배치 처리**: 다중 이미지 동시 분석
- **리소스 관리**: GPU/CPU 효율적 활용

**데이터 보안**
